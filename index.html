Nim Tutorial

Nim is a compiled, statically typed programming language that allows quick compilation and easy execution. Code can be written in a file with the .nim extension and executed either through compilation or interactively via a REPL.

1. Entry Point and Code Execution

Nim executes top-level statements directly without requiring a specific entry-point function:

echo "Hello, Nim!"

Compile and run Nim code easily from the command line:

nim c -r hello.nim

2. Input and Output (Simple I/O)

Standard output uses echo or stdout.write:

echo "Enter your name:"
let name = readLine(stdin)
echo "Hello, ", name

Printing without a newline:

stdout.write("This line won't end with a newline.")

3. Data Types and Operators

  • Primitive Data Types: int, float, bool, char, string
  • Operators:
    • Arithmetic: +, , , /, mod
    • Logical: and, or, not
    • Relational: ==, !=, <, >, <=, >=

Example:

let sum = 10 + 5
echo sum

______________________________________________________________________________________________________

4. String Handling

Strings are immutable and efficiently handled:

Concatenation and slicing:

let greeting = "Hello, " & "World!"
echo greeting

echo greeting[0..4]  # "Hello"

String repetition:

let stars = repeat("*", 5)
echo stars  # "*****"

______________________________________________________________________________________________________

5. Built-in Data Structures

  • Arrays (fixed-size):
var nums: array[3, int] = [1, 2, 3]
  • Sequences (dynamic arrays):
var seqs = @[1, 2, 3]
seqs.add(4)
  • Sets:
var vowels: set[char] = {'a', 'e', 'i', 'o', 'u'}
  • Hash Tables:
import tables
var ages = {"Alice": 30, "Bob": 25}.toTable
echo ages["Alice"]

______________________________________________________________________________________________________

6. Control Structures

Conditional logic:

let x = 10
if x > 0:
  echo "Positive"
elif x == 0:
  echo "Zero"
else:
  echo "Negative"

Loops:

# For loop
for i in 1..5:
  echo i

# While loop
var i = 0
while i < 5:
  echo i
  inc(i)

______________________________________________________________________________________________________

7. Functions (scoping, closures)

Basic function:

proc greet(name: string): string =
  "Hello, " & name

echo greet("Nim")

Closure:

proc makeAdder(x: int): proc(y: int): int =
  return proc(y: int): int = x + y

let addFive = makeAdder(5)
echo addFive(10)  # 15

______________________________________________________________________________________________________

8. User-Defined Types

Objects (similar to structs or records):

type
  Person = object
    name: string
    age: int

var p = Person(name: "Emma", age: 22)
echo p.name, " is ", p.age

______________________________________________________________________________________________________

9. Modularization

Defining and using modules:

Module (greetings.nim):

proc sayHello*() =
  echo "Hello from module"

Using the module:

import greetings
sayHello()

______________________________________________________________________________________________________

10. Notable Features and Advanced Concepts

  • Memory Management: Nim uses garbage collection by default, with options for manual management via ARC/ORC.
  • Metaprogramming (Macros and Templates):
template debug(msg: string) =
  echo "[DEBUG] ", msg

debug("Testing")
  • Interoperability with C:
{.passC: "-lm".}  # Linking to C math library

+Practical Use Case: Virtual Animal Clinic with Method Dispatch

type
  Animal = object of RootObj
  Cat = object of Animal
  Dog = object of Animal

method speak(a: Animal) =
  echo "Some animal sound"

method speak(c: Cat) =
  echo "Meow"

method speak(d: Dog) =
  echo "Woof"

template showWelcome() =
  when defined(windows):
    echo "Welcome to the Windows Animal Hospital!"
  else:
    echo "Welcome to the Animal Hospital!"

block animalClinic:
  echo "== Animal Clinic Open =="
  showWelcome()

  let animals: seq[Animal] = @[Cat(), Dog(), Dog()]

  for a in animals:
    speak a

  echo "Too many patients today. Closing early."
  break animalClinic

  echo "This message won't be shown."

+Language Comparison: Nim vs Python

FeatureNimPython
TypingStatic, strongDynamic, strong
PerformanceCompiled (fast execution)Interpreted (slower)

CompilationRequired (nim c file.nim)Direct execution
