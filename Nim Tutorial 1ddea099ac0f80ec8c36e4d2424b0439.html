Nim Tutorial

Nim is a compiled, statically typed programming language that allows quick compilation and easy execution. Code can be written in a file with the .nim extension and executed either through compilation or interactively via a REPL.

1. Entry Point and Code Execution

2. Input and Output (Simple I/O)

Nim provides straightforward syntax for both standard input and output. It uses built-in procedures such as echo, readLine, and stdout.write.

  • Output Example:
    echo "This is a message printed to the console."
  • Input Example:
    echo "Enter your name:"
let name = readLine(stdin)
echo "Hello, ", name
  • Using ****stdout.write:
    stdout.write("This is printed without a newline.")

These basic I/O operations allow you to interact with users through the console.

3. Primitive Data Types and Operators

Nim supports a variety of primitive data types and operators that are commonly used in modern programming.

  • Primitive Data Types:
    • Integer types: int, int8, int16, int32, int64
    • Floating-point types: float, float32, float64
    • Boolean: bool
    • Character: char
    • String: string
  • Type System:
    • Nim is statically typed: types are checked at compile time.
    • Strongly typed: implicit type conversions are not allowed unless explicitly defined.
  • Operators:
    • Arithmetic: +, , , /, mod
    • Relational: ==, !=, <, >, <=, >=
    • Logical: and, or, not
    • Compound assignment: +=, =, etc.

These features provide strong type safety and reduce runtime errors.

4. String Handling

Strings are immutable sequences of characters in Nim and are handled efficiently. They support a variety of operations.

  • Creating and Concatenating Strings:
    let name = "Nim"
let message = "Hello, " & name
echo message
  • Slicing:
    let sub = message[0..4]  # Extracts "Hello"
  • Repetition:
    let stars = repeat("*", 5)  # Produces "*****"
  • Immutability:
    • Strings in Nim are immutable. Any modification results in a new string being created.

Understanding string behavior is crucial for tasks like text processing and memory-efficient applications.

5. Built-in Data Structures

Nim provides several built-in data structures for various tasks, including arrays, sequences, sets, and tables.

  • Arrays:
    var nums: array[3, int] = [1, 2, 3]
  • Sequences (Resizable Arrays):
    var seqs = @[1, 2, 3]
seqs.add(4)
  • Sets:
    var alphabet: set[char] = {'a', 'b', 'c'}
  • Hash Tables (Dictionaries):
    import tables
var info = {"language": "Nim", "type": "compiled"}.toTable
echo info["language"]
  • Destructuring and Pattern Matching:
    let (a, b) = (10, 20)
echo a, ", ", b

These structures allow flexible and powerful data organization in programs.

6. Control Structures

Control structures in Nim include conditional branching, pattern matching, and loop constructs.

  • If-Elif-Else:
    if x > 0:
  echo "Positive"
elif x == 0:
  echo "Zero"
else:
  echo "Negative"
  • Case Statement:
    case x
of 1: echo "One"
of 2: echo "Two"
else: echo "Other"
  • Loops:
    • For loop:
      for i in 0..4:
  echo i
    • While loop:
      var i = 0
while i < 5:
  echo i
  inc(i)
  • Loop Control:
    • break, continue, and return are all supported in appropriate contexts.

These structures form the foundation of structured programming in Nim.

7. Functions and Scoping

Functions in Nim are first-class citizens and support various advanced features like default parameters, varargs, and closures.

  • Defining and Calling Functions:
    proc greet(name: string): string =
  return "Hello, " & name

echo greet("Nim")
  • Default Parameters and Varargs:
    proc say(msg: string = "Hi") =
  echo msg

proc printAll(args: varargs[string]) =
  for a in args:
    echo a
  • Anonymous and Higher-order Functions:
    let double = proc(x: int): int = x * 2
echo double(4)
  • Closures and Scope:
    • Nim uses lexical scoping.
    • Inner functions can capture variables from enclosing scopes.

These features allow for flexible functional-style programming in Nim.

8. User-Defined Types

Nim supports defining custom types using object, similar to structs in C or records in Pascal.

  • Defining and Using Objects:
    type Person = object
  name: string
  age: int

var p = Person(name: "Alice", age: 30)
echo p.name, " is ", p.age, " years old."

Custom types improve code organization and are essential for larger programs.

9. Modularization

Nim encourages code modularity by supporting modules, imports, and package organization.

  • Creating a Module:
    # greet.nim
proc hello() = echo "Hello from module"
  • Using a Module:
    import greet
hello()
  • Organizing Projects:
    • Use folders and files with consistent naming.
    • Export procedures using symbol, e.g., proc hello*().

Modular programming promotes code reuse and maintainability.

10. Notable Features and Advanced Concepts

Nim includes advanced features such as manual memory management, macros, and interoperability with C.

  • Memory Management:
    • Nim uses garbage collection (default).
    • You can switch to ARC/ORC GC using -gc:arc or -gc:orc for more control.
  • Templates and Macros:
    template sayHi(name: string): untyped =
  echo "Hi, ", name

sayHi("Julia")
    • Macros provide compile-time metaprogramming capabilities.
  • C Interoperability:
    {.passC: "-lm".}  # Link to C's math library

These features make Nim suitable for both high-level scripting and low-level systems programming.

+Practical Use Case: Number Guessing Game

This interactive number guessing game demonstrates Nim's control structures, user input, random number generation, type conversion, and looping mechanisms.

import random

randomize()  # Initializes the random number generator
let secret = rand(10)  # Generates a random number between 0 and 10
var guess: int

while true:
  echo "Guess the number (0-10):"
  try:
    guess = parseInt(readLine(stdin))
    if guess == secret:
      echo "Correct! The number was ", secret
      break
    elif guess < secret:
      echo "Too low!"
    else:
      echo "Too high!"
  except ValueError:
    echo "Please enter a valid integer."

This small game is easy to understand and fun to play — perfect for beginners learning flow control and user interaction in Nim.
